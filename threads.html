  — 09/16/2021: Note -- Threads --

  A thread does 'hanging' -- concurrent programming; See also ; spawn computing.
  We'll start off by listing all the terms associated with it:

  ————— semaphore
  ————— spinlock
  ————— atomic
  ————— mutex
  ————— volatile


  A lock (or mutex) is a mechanism that enforces a limitation to a resource,
  especially when there are many threads of execution.
  A lock is designed to enforce a mutual exclusion concurrency policy...
  The simplest type of lock being; a binary semaphore. A semaphore is just
  an abstract data type in laymans terms. A spinlock, handled in a for-loop,
  will wait for a lock until it becomes available again.
  The atomic keyword refers to a cpu property used to manipulate an object
  without using a mutex lock. The volatile qualifier is used to tell the compiler that
  the value of said-code may change at any time.
